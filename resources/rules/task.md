# 任务管理

## ID 格式规范

- **基本格式**: `[任务描述]-[编号]`，如"用户认证功能开发-001"
- **层级格式**: `[主任务]-[主编号].[子编号].[孙编号]`
- **编号规则**: 主任务 3 位数字(001-999)，子任务点号分隔
- **描述规范**: 不超过 20 字符，动词开头，包含关键词

**示例**:

```
用户管理系统开发-001
  用户认证功能开发-001.001
    登录接口实现-001.001.001
    注册接口实现-001.001.002
```

## 管理要求

- **唯一性**: 项目范围内唯一
- **一致性**: 格式统一，避免混用
- **可追溯**: 可追溯到原始需求
- **可扩展**: 支持未来扩展需求
